# 云脑诊疗平台 - 患者端拓展功能与部署说明

## 一、已完成内容

### 1. AI 流式响应

患者端“AI健康顾问”已改为逐字展示 AI 回答：

1. 用户发送健康问题；
2. 前端调用后端 AI 接口获取回答；
3. Pinia 将回答按字符逐步写入消息；
4. 页面展示闪烁光标和逐字生成效果；
5. 生成结束后再显示相关问题和推荐科室。

当前接口仍兼容普通 JSON 响应。后续后端增加 SSE 或 WebSocket 接口时，可以保留页面和 Pinia 状态，只替换 `src/stores/ai.ts` 中的数据接收部分。

Nginx 已预留 `/api/ai/stream/` 流式代理，并关闭代理缓冲。

### 2. Pinia 模块化状态管理

患者端状态已按业务拆分：

| 文件 | 管理内容 |
|------|----------|
| `src/stores/auth.ts` | 患者登录状态、患者个人资料、Token |
| `src/stores/doctors.ts` | 科室、医生、当前选择科室 |
| `src/stores/registrations.ts` | 挂号记录、挂号状态筛选 |
| `src/stores/prescriptions.ts` | 处方记录、分页、已发药与待发药统计 |
| `src/stores/ai.ts` | AI会话、流式文本、推荐科室和相关问题 |

已接入页面：

- AI健康顾问；
- 医生列表；
- 挂号记录；
- 我的处方。

### 3. 前后端分离部署

患者端支持两种构建方式。

#### 微信小程序

```bash
npm run build:mp-weixin
```

构建目录：

```text
dist/build/mp-weixin
```

微信开发者工具导入该目录即可。

#### H5 + Nginx

```bash
npm run build:h5
```

构建目录通常为：

```text
dist/build/h5
```

把构建结果复制到：

```text
/usr/share/nginx/html/cloud-brain-patient
```

然后使用项目根目录中的：

```text
nginx-patient.conf
```

Nginx 会把 `/api` 请求转发到统一后端：

```text
http://127.0.0.1:8080
```

## 二、本地开发

微信小程序开发：

```bash
npm run dev:mp-weixin
```

H5开发：

```bash
npm run dev:h5
```

H5开发端口为：

```text
http://localhost:5177
```

## 三、重要说明

- 前端不直接连接数据库；
- 患者端、医生端、管理端、药房端和检验科端统一请求后端 `localhost:8080`；
- 真正的 SSE 需要后端提供持续输出接口；当前患者端已经完成逐字展示、消息状态和 Nginx 流式代理准备。
