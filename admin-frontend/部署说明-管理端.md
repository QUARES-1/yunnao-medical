# 云脑诊疗平台管理端部署说明

## 1. 本地开发

```bash
npm install
npm run dev
```

访问：`http://127.0.0.1:5174`

开发环境通过 `vite.config.ts` 将 `/api` 和 `/upload` 代理到后端 `http://localhost:8080`。

## 2. 生产打包

```bash
npm run build
```

打包完成后会生成 `dist/` 文件夹。生产部署时只需要把 `dist/` 中的静态文件放到 Nginx 静态目录。

## 3. Nginx 双端分离部署

示例配置文件：`nginx-admin.conf`

推荐目录：

```text
/usr/share/nginx/html/cloud-brain-admin
```

部署步骤：

1. 将 `dist/` 内容复制到 `/usr/share/nginx/html/cloud-brain-admin/`。
2. 将 `nginx-admin.conf` 放到 Nginx 配置目录并启用。
3. 确保后端 Spring Boot 已启动在 `8080` 端口。
4. 重载 Nginx：`nginx -s reload`。

## 4. 拓展功能说明

- AI 流式响应：管理端“AI运营中心”点击“生成运营报告”后，通过 SSE 逐字展示 AI 生成内容。
- Pinia 状态管理：`src/stores/auth.ts` 管理登录态，`src/stores/ai.ts` 管理 AI 运营数据，`src/stores/medical.ts` 管理科室、医生、药品和统计快照。
- 双端分离部署：前端作为 Nginx 静态资源运行，接口统一通过 `/api` 反向代理到后端。
