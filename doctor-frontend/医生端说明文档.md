# 云脑诊疗平台——医生端说明文档

## 一、项目概述

云脑诊疗平台医生端是一套基于 Vue 3 与 Spring Boot 的 Web 诊疗工作站，供医生完成患者接诊、病历书写、检查开立、处方管理和历史诊疗记录查询等工作。

系统已与患者端、药房端和检验科业务数据打通，并通过 JWT 进行身份认证。医生只能查看和操作挂号给自己的患者，避免不同医生之间发生患者数据串用。

- 文档版本：v2.0
- 更新时间：2026 年 6 月
- 当前阶段：传统业务模块开发，不接入 AI

## 二、功能清单

### 1. 登录与注册

- 支持医生账号、密码登录。
- 支持医生自助注册。
- 登录成功后自动保存 JWT Token。
- 未登录访问业务页面时自动跳转登录页。
- Token 失效后自动清理登录状态并返回登录页。
- 支持“记住账号”和演示账号一键填充。
- 登录密码通过 JSON 请求体传输，不再放入 URL。
- 数据库中的医生密码使用 BCrypt 加密存储。
- 旧版明文演示账号首次成功登录后会自动升级为 BCrypt 密文。

### 2. 今日工作台

- 只显示当前登录医生今天的挂号患者。
- 展示患者姓名、预约编号、科室、就诊日期、时间段和当前状态。
- 支持按患者姓名搜索。
- 支持按状态筛选。
- 展示今日患者、待就诊、就诊中和已完成数量。
- 支持手动刷新患者队列。
- 待就诊患者可点击“开始看诊”。
- 就诊中患者可点击“继续看诊”。
- 已就诊患者可查看诊疗详情。
- 不属于当前医生的患者不会显示，也不能通过手动修改网址访问。

### 3. 患者看诊工作区

看诊页面由患者资料、病历记录、检查管理和处方管理组成。

#### 3.1 患者资料

- 患者姓名
- 性别
- 年龄
- 联系电话
- 挂号日期
- 就诊时间段
- 就诊科室
- 过敏史

有明确过敏史时使用醒目的红色标签显示，帮助医生在开方前及时识别风险。

#### 3.2 病历记录

- 填写主诉、现病史、既往史和体格检查。
- 填写诊断结果和治疗意见。
- 同一挂号只对应一份病历，重复保存时自动更新。
- 完成看诊前必须保存病历。
- 主诉、诊断结果和治疗意见为完成看诊所需的核心字段。
- 待就诊患者必须先从工作台点击“开始看诊”才能编辑。
- 已就诊和已取消记录自动进入只读模式。

#### 3.3 检查与检验

- 展示当前挂号已经开立的检查记录。
- 展示项目名称、类型、状态、检查结果和开单时间。
- 支持按照“检查”或“检验”筛选项目。
- 支持为当前患者开立检查或检验。
- 防止重复开立同一项待检查项目。
- 待检查项目支持撤销。
- 已完成或已撤销项目不能重复撤销。
- 支持手动刷新检查结果。
- 出现新的已完成结果时显示提醒。

#### 3.4 处方管理

- 支持搜索药品。
- 展示药品名称、规格、单位、单价和库存。
- 支持添加多种药品。
- 支持填写数量和用法用量。
- 前端限制开药数量不能超过当前库存。
- 后端重新校验药品、数量、价格和库存。
- 处方总金额由后端根据数据库价格重新计算，不能由前端伪造。
- 创建处方后自动扣减库存。
- 待发药处方支持撤销。
- 撤销处方时自动返还药品库存。
- 已撤销处方保留在数据库中用于审计，但不会继续显示在医生端有效处方列表。
- 支持打印格式化处方单。

#### 3.5 完成看诊

- 只有“就诊中”的挂号可以完成看诊。
- 完成前必须保存完整病历。
- 完成后挂号状态变为“已就诊”。
- 完成后自动返回工作台。
- 已完成的诊疗资料自动变为只读。

### 4. 历史诊疗记录

- 只显示当前医生的已就诊和已取消记录。
- 不再将今天的待就诊、就诊中患者混入历史记录。
- 支持服务端分页。
- 支持按患者姓名进行服务端搜索。
- 支持按已就诊、已取消状态进行服务端筛选。
- 搜索范围覆盖全部历史数据，不局限于当前页。
- 支持查看历史病历、检查和处方详情。

### 5. 个人中心

- 查看登录账号、医生姓名、联系电话、职称和所属科室。
- 修改医生姓名、联系电话、职称和科室。
- 修改擅长领域和个人简介。
- 展示资料完整度。
- 支持上传 JPG、PNG、WebP 格式的医生头像。
- 头像大小限制为 2 MB。
- 头像上传接口需要医生身份认证。
- 支持通过原密码修改登录密码。
- 新密码长度不能少于 6 位，且不能与原密码相同。

## 三、权限与数据安全

### 1. 身份认证

- 认证方式：JWT。
- Token 由前端保存在 `localStorage` 中。
- 请求时通过请求头中的 `token` 字段携带。

### 2. 医生数据隔离

当前登录医生只能访问挂号给自己的患者，包括：

- 今日挂号列表
- 历史诊疗记录
- 挂号详情
- 电子病历
- 检查检验记录
- 处方记录

即使手动修改挂号编号或接口地址，后端也会拒绝访问其他医生的患者。

### 3. 业务状态校验

- 只有就诊中的患者可以编辑病历、开检查和开处方。
- 只有待检查项目可以撤销。
- 只有待发药处方可以撤销。
- 只有就诊中的挂号可以完成看诊。
- 完成看诊前必须存在完整病历。

### 4. 密码安全

- 医生密码使用 BCrypt 单向加密。
- 登录接口使用 JSON 请求体传递账号密码。
- 接口返回医生信息时不会返回密码字段。

## 八、主要 API

### 8.1 医生账号与个人信息

| 方法 | 接口 | 说明 |
|---|---|---|
| POST | `/api/doctor/login` | 医生登录，JSON 请求体 |
| POST | `/api/doctor/register` | 医生注册，JSON 请求体 |
| GET | `/api/doctor/info` | 获取当前医生信息 |
| PUT | `/api/doctor/update` | 修改医生个人信息 |
| PUT | `/api/doctor/change-password` | 修改当前医生密码 |
| POST | `/api/file/upload` | 上传医生头像 |

登录请求示例：

```json
{
  "username": "doctor01",
  "password": "123456"
}
```

### 8.2 挂号与看诊

| 方法 | 接口 | 说明 |
|---|---|---|
| GET | `/api/registration/doctor/today` | 当前医生今日患者 |
| GET | `/api/registration/doctor/list` | 当前医生历史记录，支持分页、搜索和状态筛选 |
| GET | `/api/registration/detail/{id}` | 挂号详情与患者临床资料 |
| PUT | `/api/registration/start/{id}` | 开始看诊 |
| PUT | `/api/registration/complete/{id}` | 完成看诊 |

历史查询参数：

| 参数 | 说明 |
|---|---|
| `page` | 页码，从 1 开始 |
| `size` | 每页数量 |
| `keyword` | 患者姓名关键词 |
| `status` | 已就诊或已取消 |

### 8.3 电子病历

| 方法 | 接口 | 说明 |
|---|---|---|
| POST | `/api/medical-record/save` | 保存或更新病历 |
| GET | `/api/medical-record/registration/{regId}` | 根据挂号查询病历 |
| GET | `/api/medical-record/detail/{id}` | 病历详情 |

### 8.4 检查检验

| 方法 | 接口 | 说明 |
|---|---|---|
| GET | `/api/examination/item/list` | 检查项目列表 |
| POST | `/api/examination/create` | 开立检查 |
| PUT | `/api/examination/cancel/{id}` | 撤销待检查项目 |
| GET | `/api/examination/registration/{regId}` | 当前挂号检查记录 |
| GET | `/api/examination/detail/{id}` | 检查详情 |

### 8.5 处方与药品

| 方法 | 接口 | 说明 |
|---|---|---|
| GET | `/api/medicine/list` | 药品列表与搜索 |
| POST | `/api/prescription/create` | 开具处方并扣减库存 |
| PUT | `/api/prescription/cancel/{id}` | 撤销待发药处方并返还库存 |
| GET | `/api/prescription/registration/{regId}` | 当前挂号有效处方 |
| GET | `/api/prescription/detail/{id}` | 处方详情 |

## 九、数据库说明

主要数据表：

| 表名 | 说明 |
|---|---|
| `doctor` | 医生账号、科室、职称、头像和专业资料 |
| `patient` | 患者资料、联系方式和过敏史 |
| `department` | 科室字典 |
| `registration` | 挂号记录与看诊状态 |
| `medical_record` | 电子病历 |
| `examination_item` | 检查检验项目字典 |
| `examination` | 检查检验申请与结果 |
| `medicine` | 药品字典、价格与库存 |
| `prescription` | 处方、金额和发药状态 |

处方和检查撤销时采用“修改状态”而不是直接删除记录，从而保留业务审计信息。

## 十、核心业务流程

```text
患者预约挂号
    ↓
医生工作台出现待就诊患者
    ↓
医生点击开始看诊
    ↓
填写并保存电子病历
    ↓
按需开立检查/检验
    ↓
按需开具处方并扣减库存
    ↓
医生确认并完成看诊
    ↓
记录进入历史诊疗档案
```

异常处理：

- 误开检查：在检查完成前撤销。
- 误开处方：在药房发药前撤销，库存自动返还。
- 已完成检查和已发药处方不能由医生撤销。
- 已撤销记录保留在数据库中，但不作为有效处方显示。

## 十二、注意事项

1. 必须先启动 MySQL 和后端，再启动医生端前端。
2. 前端默认访问 `http://localhost:8080`。
3. 如果页面没有显示最新效果，请按 `Ctrl + F5` 强制刷新。
4. 修改数据库连接信息后需要重新启动后端。
5. 不要将 `node_modules` 放入共享压缩包或提交到 Git。
6. 不要分享真实数据库密码、JWT 密钥或微信 AppSecret。
7. 完成看诊后资料进入只读模式，请在操作前确认病历已保存。
8. 已撤销处方不会显示在医生端有效处方列表，但仍保存在数据库中。
