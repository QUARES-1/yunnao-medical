# 云脑诊疗平台 - 患者端前端说明文档

## 一、项目概述

本项目是“东软智慧云脑诊疗平台”的患者端微信小程序，面向患者提供科室查询、医生筛选、在线挂号、挂号记录管理、微信登录与个人资料维护等功能。

当前阶段仅实现传统医疗业务模块，不接入 AI 功能，重点跑通“选择科室 → 选择医生 → 查看医生 → 选择排班 → 提交挂号 → 查看记录”的完整业务流程。

开发完成时间：2026年6月

---

## 二、功能清单

患者端共包含 9 个页面，覆盖患者管理与在线挂号的核心功能。

### 1. 首页

- 展示云脑诊疗品牌信息、快捷搜索和健康提示
- 提供“预约挂号”和“我的预约”快捷入口
- 展示 6 个热门科室，采用三列等宽卡片布局
- 点击热门科室可直接查看该科室医生
- 点击“查看全部”进入完整科室列表
- 首页右上角显示患者真实头像或姓名首字
- 支持转发给朋友和分享到朋友圈

### 2. 全部科室页

- 展示数据库中的全部科室
- 支持按科室名称或科室描述搜索
- 每个科室显示名称、简介和图标
- 点击科室后进入该科室的医生列表

### 3. 医生列表页

- 展示医生姓名、科室、职称和擅长领域
- 支持按医生姓名、科室、职称和擅长领域搜索
- 支持按科室筛选医生
- 支持按职称筛选医生，例如主任医师、副主任医师、主治医师
- 职称选项根据当前医生数据自动生成
- 显示当前筛选条件下的医生数量
- 点击医生卡片进入医生主页

### 4. 医生主页

- 展示医生姓名、职称、所属科室、擅长领域和个人简介
- 提供“立即预约”入口
- 点击后进入确认预约页面

### 5. 确认预约页

- 展示当前选择的医生与科室
- 提供未来 7 天的可预约日期
- 提供上午、下午两个就诊时段
- 显示每个时段的具体剩余号源，例如“余 15 号”
- 已约满时段不可选择
- 未登录患者提交挂号时会提示先登录
- 挂号成功后自动进入挂号详情页

### 6. 挂号记录页

- 展示当前患者的全部挂号记录
- 支持按状态筛选：全部、待就诊、就诊中、已就诊、已取消
- “全部”展示其他状态记录的总和
- 每条记录展示科室、医生、日期、时段、状态和预约编号
- 点击记录进入挂号详情页

### 7. 挂号详情页

- 突出展示就诊日期和上午/下午时段
- 展示就诊状态、科室、接诊医生、患者姓名和预约编号
- 待就诊状态支持取消预约
- 提供“完成并返回首页”按钮
- 取消成功后自动刷新记录状态

### 8. 个人中心页

- 支持微信小程序真实登录
- 展示患者头像、昵称和手机号
- 提供挂号记录入口
- 预留电子病历、检查报告和我的处方入口
- 展示健康服务热线

### 9. 编辑个人资料页

- 使用微信原生头像选择能力选择真实头像
- 使用微信昵称输入能力填写真实昵称
- 头像上传至后端并持久化保存
- 昵称保存至患者数据库
- 保存后个人中心与首页头像同步更新

## 三、技术栈

| 类别 | 技术 |
|------|------|
| 前端框架 | Vue 3 |
| 开发语言 | TypeScript |
| 跨端框架 | uni-app |
| 构建工具 | Vite |
| 状态管理 | Pinia |
| 微信端编译 | @dcloudio/uni-mp-weixin |

## 五、运行说明

### 5.1 环境要求

- Node.js 18 或以上
- npm 9+ 或 pnpm 8+
- 微信开发者工具 Stable 版本
- Java 17
- MySQL 8.0
- 后端服务运行在 `http://localhost:8080`

### 5.2 启动后端

1. 进入后端项目目录：`cloud-brain-diagnosis`
2. 启动 MySQL
3. 创建数据库：`cloud_brain_diagnosis`
4. 修改 `src/main/resources/application.yaml` 中的数据库账号和密码
5. 真实微信登录时，通过环境变量配置 AppSecret，不要将 AppSecret 写入代码
6. 启动后端：

```powershell
.\mvnw.cmd spring-boot:run
```

7. 后端 API 文档地址：`http://localhost:8080/doc.html`

### 5.3 安装患者端依赖

进入患者端项目目录：

```bash
cd cloud-brain-patient
```

使用 npm：

```bash
npm install
```

或使用 pnpm：

```bash
pnpm install
```

`node_modules` 不需要在电脑之间复制，可根据 `package.json` 和锁文件重新安装。

### 5.4 类型检查与编译

```bash
npm run type-check
npm run build:mp-weixin
```

编译成功后生成：

```text
dist/build/mp-weixin
```

### 5.5 导入微信开发者工具

1. 打开微信开发者工具
2. 点击“导入项目”
3. 项目目录选择 `dist/build/mp-weixin`
4. 选择项目已登记的小程序 AppID
5. 点击“导入”并执行“编译”

如果只需要预览，也可以直接导入压缩包中已有的 `dist/build/mp-weixin`，无需安装依赖。

## 六、API 接口说明

患者端默认后端地址为 `http://localhost:8080`。

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/patient/wx-login | POST | 微信登录，返回患者 token |
| /api/patient/info | GET | 获取当前患者资料 |
| /api/patient/update | PUT | 修改患者昵称、头像等资料 |
| /api/file/upload | POST | 上传患者头像 |
| /api/department/list | GET | 获取全部科室 |
| /api/doctor/list | GET | 获取医生列表，支持科室筛选 |
| /api/doctor/detail/{id} | GET | 获取医生详情 |
| /api/doctor/{id}/schedule | GET | 获取未来日期、时段与剩余号源 |
| /api/registration/create | POST | 创建挂号记录 |
| /api/registration/patient/list | GET | 获取患者挂号记录，支持状态筛选 |
| /api/registration/detail/{id} | GET | 获取挂号详情 |
| /api/registration/cancel/{id} | PUT | 取消待就诊挂号 |

需要认证的接口在请求头中携带：

```text

Authorization: Bearer <JWT token>
```

## 七、数据库说明

数据库名：`cloud_brain_diagnosis`，建议字符集为 `utf8mb4`。

患者端当前主要使用以下数据表：

| 表名 | 说明 |
|------|------|
| patient | 患者账号、微信 openid、昵称、头像和个人资料 |
| department | 科室信息 |
| doctor | 医生信息、职称、科室和擅长领域 |
| registration | 挂号日期、时段、患者、医生和状态 |

当前开发数据库已扩充为多科室、多医生测试数据；队友在其他电脑运行时，需要单独导入测试数据或使用自己的数据库数据。

## 八、核心业务流程

```text
微信登录
   ↓
首页选择热门科室 / 查看全部科室
   ↓
选择科室
   ↓
按职称筛选医生
   ↓
查看医生主页
   ↓
选择日期和上午/下午时段
   ↓
查看具体剩余号源并提交预约
   ↓
查看挂号详情
   ↓
在挂号记录中跟踪待就诊、就诊中、已就诊或已取消状态
```

## 九、注意事项

1. 数据库密码必须修改为当前电脑自己的 MySQL 密码，不要直接复制其他成员的密码。
2. AppSecret 属于敏感信息，不得写入前端、压缩包、GitHub 或公共聊天记录。
3. 使用正式 AppID 调试时，小程序管理员需要将队友微信号添加为开发成员。
4. 微信开发者工具模拟器访问 `localhost:8080` 时，后端必须在同一台电脑运行。
5. 真机预览时，手机中的 `localhost` 不是电脑，需要将 API 地址改为电脑的局域网 IP，并保证手机与电脑连接同一网络。
6. 页面显示旧内容时，执行“清缓存 → 全部清除”，关闭重复的开发者工具窗口后重新编译。
7. 患者头像默认上传到后端配置的 `D:/upload/` 目录，需要保证该目录可写。
8. “浮窗、添加到桌面、在电脑打开”等菜单由真实微信客户端提供，开发者工具无法完整模拟。
9. 当前阶段为传统业务基础版，不接入 AI 诊断、AI 问答或智能推荐。
