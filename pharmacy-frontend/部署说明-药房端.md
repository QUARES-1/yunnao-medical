# 云脑诊疗平台 - 药房端拓展功能与部署说明

## 一、已完成的拓展任务

### 1. AI 流式响应展示

药房端在“AI库存预测”页面新增了“AI流式生成报告”功能。

实现效果：

- 用户先生成或加载一份库存预测；
- 点击“AI流式生成报告”；
- 页面会逐字输出库存预测报告，包括总体结论、优先采购建议、风险提醒、药房执行建议；
- 当前为前端模拟 SSE / WebSocket 的逐字展示效果，后续如果后端提供真实 SSE 或 WebSocket 接口，只需要替换 Pinia store 中的流式数据来源即可。

相关文件：

- `src/views/ForecastView.vue`
- `src/stores/pharmacy.ts`

### 2. Pinia 状态管理优化

新增药房端业务状态模块 `src/stores/pharmacy.ts`，用于统一管理：

- 待发药处方；
- 已发药处方；
- 药品库存；
- 低库存药品；
- AI库存预测列表；
- AI流式生成文本状态。

这样后续处方队列、库存查询、AI预测页面都可以复用同一份状态，避免不同页面重复请求和状态混乱。

### 3. 双端分离部署

新增 Nginx 部署配置：

- `nginx-pharmacy.conf`

部署方式：

1. 执行前端打包：

```bash
npm run build
```

2. 将 `dist` 目录复制到服务器：

```text
/usr/share/nginx/html/cloud-brain-pharmacy
```

3. 将 `nginx-pharmacy.conf` 放入 Nginx 配置目录，例如：

```text
/etc/nginx/conf.d/cloud-brain-pharmacy.conf
```

4. 重载 Nginx：

```bash
nginx -s reload
```

## 二、本地运行方式

### 1. 启动后端

后端统一使用：

```text
http://localhost:8080
```

### 2. 启动药房端

进入药房端目录：

```bash
cd cloud-brain-pharmacy
npm install
npm run dev
```

默认访问地址：

```text
http://127.0.0.1:5175
```

## 三、接口说明

药房端通过 Vite 代理访问后端：

```text
/api -> http://localhost:8080/api
```

前端不直接连接数据库，所有数据都通过后端 RESTful API 获取。

## 四、说明

当前“AI流式生成报告”已经具备前端逐字展示效果，适合实训答辩演示。若后端后续提供真实 AI SSE 接口，可以保留页面样式和交互，只替换 `src/stores/pharmacy.ts` 中的 `streamForecastReport` 实现。
